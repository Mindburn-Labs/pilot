# Pilot Capability Status

Source of truth: `packages/shared/src/capabilities/index.ts`.

Registry revision: `2026-05-06T00:00:00.000Z`.

Production-ready capabilities: `0/18`.

This Gate 0 status document is intentionally conservative. Pilot must not claim production autonomous startup OS readiness until a capability is marked `production_ready` by the registry and includes passing eval metadata with an evidence reference.

Eval readiness is exposed at `GET /api/evals/readiness`. That endpoint is the workspace-scoped inventory for missing real evals, required external resources, required HELM policies, evidence requirements, audit requirements, and recent eval runs. It reports the default executor as `control_plane_proof_check`; production promotion requires passed durable runs with server-trusted `metadata.executionMode = "real_external_eval"`. Manual eval-run metadata cannot forge that mode. `/api/evals/execute` routes `real_external_eval` through server-owned runners; unsupported scenarios fail closed, and the current HELM Governance, Decision Court Governed Model, PMF Discovery opportunity-scoring, Safe Computer/Sandbox, and YC Logged-In Browser Extraction runners only pass after verifying durable receipt, court, opportunity, computer, or browser rows through linked `evidence_items` and `audit_log` rows.

Operational production gates are tracked in [`docs/production-readiness-checklist.md`](production-readiness-checklist.md). That checklist is not a second source of truth; it translates the registry into merge and eval requirements for the remaining PRs.
